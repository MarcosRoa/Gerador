🧠 Loterias V.7.0 SEGURANÇA

Documentação de segurança do sistema

🔒 Segurança do Sistema v7.0

🛡️ Camadas de Segurança

🔐 Autenticação Firebase Implementado
Login via Google, Facebook e E-mail/Senha.
Tokens JWT expiram após 1 hora.
Domínios autorizados configurados no Firebase Console.
🛡️ RBAC (Role-Based Access Control) Implementado
Papéis: guest, free, pro, admin, super_admin.
Permissões granulares para cada ação do sistema.
Verificação em cada endpoint da API.
⏱️ Rate Limiting Implementado
30 requisições por minuto (padrão).
5 requisições por 10 segundos (endpoint /generate).
5 tentativas de login por minuto.
Protege contra ataques de força bruta e DDoS.
🤖 Bot Detection Implementado
Detecta bots baseado em User-Agent e headers.
Score de 0-100 quanto maior mais suspeito.
Bloqueia requisições com score > 60.
⚠️ Anomaly Detection Implementado
Monitora comportamento do usuário em janela de 5 minutos.
Detecta: muitas ações, alta frequência, múltiplos usuários/IP.
Gera alertas no log de auditoria.
📜 Audit Logger Implementado
Registra todas as ações importantes (login, geração, compra).
Salva IP, User-Agent, timestamp e detalhes.
Opcional: persistência no banco de dados.
⚡ Transações Atômicas (RPC) Implementado
Operações TUDO ou NADA via RPC do Supabase.
SELECT FOR UPDATE para pessimistic locking.
Idempotência via idempotency_key.
🔑 PRO fixo por e-mail Implementado
E-mails na lista de PRO sempre têm plano ativo.
Créditos resetados para 100 a cada login.
Configurável via variável de ambiente.
💳 Webhook com verificação Parcial
Verificação de assinatura HMAC-SHA256.
Idempotência para evitar duplicação.
⚠️ Mercado Pago real: configurar ACCESS_TOKEN.

📝 Recomendações para Produção

  • Ativar RLS (Row Level Security) no Supabase
  • Configurar Redis para rate limiting distribuído
  • Usar variáveis de ambiente para todas as chaves secretas
  • Implementar Webhook real com Mercado Pago
  • Adicionar 2FA para administradores
  • Realizar testes de penetração periódicos
